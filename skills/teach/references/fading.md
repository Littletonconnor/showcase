# The fading arc (worked examples to problems)

Novices learn more from studying worked examples; experts learn more from
solving. The bridge is fading: worked example, then completion problems, then
full problems. The learner level sets where each concept STARTS on that arc;
mastery moves it forward.

## Per level, within one concept beat

**novice**

1. Full worked example, annotated: every step shown, the WHY beside each step.
2. Completion checkpoint: the same shape of problem with one step blanked.
3. Only then a generation checkpoint (apply), and only if the completion held.

Hooks stay: even novices predict before teaching. What novices skip is
unguided problem-solving before any schema exists.

**intermediate**

1. Compressed worked example: the skeleton, annotations only on the non-obvious
   steps.
2. Straight to completion or trace checkpoints.
3. An apply checkpoint with a varied surface context.

**advanced**

1. Problem FIRST (apply/trace), worked example only as the reveal or on a miss.
   For experts, studying redundant guidance is worse than solving (expertise
   reversal); respect their time.
2. Interleave aggressively with adjacent concepts.

## Auto-fading across sessions

Read the mastery state before planning (get_learner_state when showcase is
present, or ask):

- prerequisite `solid`: do not re-teach it; reference it and build. Drop its
  worked examples entirely.
- prerequisite `shaky`: one completion checkpoint on it BEFORE building on it.
  If missed, remediate before the new concept (do not build on sand).
- concept `untouched` but the level is advanced: start at problems anyway.

## Signals to slow back down (unfade)

- Two misses in a row on the same concept.
- Repeated skips.
- explain-back answers that restate words without mechanism.

Response: drop back one stage (problem to completion, completion to worked
example), shrink the step size, and use a DIFFERENT example rather than
repeating the failed one.
